javaScript的Object总结

对象的定义

对象是一种复合值，将多个值以键值对的形式保存，可以通过名字访问值，也可以看做是属性的无序集合。除了可以保有自身的属性，还可以从原型对象继承属性。

对象的基本操作

对对象的操作可以理解为对对象属性的操作，对象的用法包括创建（create）、设置（set）、查找（query）、删除（delete）、检测（test）和枚举（enumerate）属性。属性名可以是包含空字符串在内的任意字符串。值可以是任意的类型，或者一个getter或setter函数。每个属性都有自身的特性：

  • 可写（writable attribute），是否可设置该属性的值。
  • 可枚举（enumerable attribute）,表明是否可以通过for/in循环。
  • 可配置（configurable attribute），是否可以删除或修改该属性。

创建对象

  • 使用对象直接量，属性可以是标识符，也可以是字符串直接量，值可以是javaScript的任意值；
  • 通过new创建对象，关键字new后面是一个函数的运行，我们称这种函数为构造函数；
  • Object.create()，Object的静态方法，生成一个继承对象；

属性的查询和设置

对象的值可以通过点(.)或方括号([])获取。方括号里面是一个字符串，而点后面必须是一个标识符。通过这种方法可以取或设置对象的值。

继承，对象具有自有属性，也有一些属性是从原型链继承而来的。对象读取属性时会找自身是否有该属性，没有该属性时会沿着原型链找，看原型对象上面是否有。当修改属性时并不会修改原型上面的属性。当访问一个对象没有的属性，会返回undefined

删除属性

用delete删除对象属性，不可删除不可配置的对象属性。实际上delete只是删除对象属性的引用，并不能删除真正的值。

检测属性

检测某个属性是否存在于对象中，常用的四种方法。

  • in运算符，可以沿原型链检测属性；
  • hasOwnProperty(),用来检测是否是对象的自有属性，无法检测继承的属性，继承的属性会返回false;
  • propertyIsEnumerable()，只能检测自身属性，且必须属性的enumerable为true才会返回true;
  • 可以通过该属性是否为undefined，来检测是否有该属性。不过无法检测属性为undefined

枚举属性

遍历对象的属性，是我们经常会用到的，通常我们会使用for/in来遍历对象的属性，for/in会遍历对象的所有可枚举的属性，包括原型链上面的，通常为了让for/in只遍历自身属性，会使用hasOwnProperty来检测。

还有Object的静态两个静态方法(keys,getOwnPropertyNames)也可以遍历对象的属性。
keys和getOwnPropertyNames都是遍历自身的属性，不会遍历原型链上的，区别在于keys只返回所有可枚举的属性，而getOwnPropertyNames不管是否可枚举会返回。

属性的getter和setter

对象处理通常用的键值对的形式来表示，还可以用一个或两个函数来表示，就是getter和setter函数，当只有setter时，表示这个属性可写，但是不可读；当只有getter时，表示这个属性只可读，不可写。在getter和setter里面this指向对象本身。getter和setter函数并不用像普通函数那样，通过关键字function，而是用set和get来配置。

属性的特性

属性包含一个名字和四个特性。四个特性包含值（value）、可写性（writable）、可枚举（enumerable）、和可配置型（configurable）。存储器属性没有值和可写性，只有set设值，和get取值，可枚举和可配置性。

  • getOwnPropertyDescriptor()，获取属性的特性，该静态方法的第一个参数为对象，第二个为对象的自有属性，针对自有属性操作，无法针对继承属性。
  • defineProperty()，设置对象的属性，并配置对象的特性，三个参数，第一个为要配置的对象，第二个为配置的属性，第三个为配置的特性。
  • defineProperties()，两个参数第一个为要配置的对象，第二个为配置多个属性的特性。

对象的三个属性

  • 原型属性

原型属性是在实例对象之前就设置好的，通过对象直接量创建的对象是将Object.prototype作为自身的原型，通过new构造器生成的对象，将构造器的prototype作为自己的原型。而通过Object.create()创建的对象是以第一个参数作为自己的原型。对象原型上面有个方法可以检测原型，Object.prototype.isPrototypeOf();
